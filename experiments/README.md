# Experiments

Use this directory for lightweight experiment manifests and registry files.
Generated model outputs belong in `outputs/` and should not be committed.

For each reported experiment, record:

- Experiment id or run name.
- Git commit.
- Config file.
- Data and feature-bundle version.
- Split directory.
- Output/checkpoint directory.
- Main validation or test metrics.

The starter registry is in `experiment_registry.csv`.
