# Method Description

## Overview

This document describes the methodology used for single-component melting point prediction in this repository.

## Model Architecture

The model is a multimodal fusion architecture that combines multiple molecular representations to predict melting points.

### Components

1. **MoLFormer Encoder**
   - SMILES-based sequential encoder
   - Captures molecular structure as a sequence
   - Uses pre-trained weights for better initialization

2. **D-MPNN Encoder**
   - Directed Message Passing Neural Network
   - Captures graph-based molecular structure
   - Processes molecular connectivity and atom/bond features

3. **XTB Features**
   - Quantum mechanical features generated by XTB
   - Provides physical and chemical properties
   - Includes energy, geometry, and electronic structure information

4. **RDKit Features**
   - Traditional molecular descriptors
   - Includes topological, constitutional, and physicochemical properties

5. **Dynamic Readout Fusion Head**
   - Residual boosting architecture
   - Dynamically weights different modalities
   - Combines features from all encoders

## Training Strategy

### Loss Function
- Huber loss for robust regression
- Balances between MAE and MSE

### Optimization
- AdamW optimizer with different learning rates for different components
- Learning rate scheduling with warmup and decay
- Gradient clipping to prevent exploding gradients

### Training Stages
1. **Warmup** (frozen BERT)
   - Train only the fusion head and D-MPNN
   - Allows the model to learn basic feature combinations

2. **Joint Training**
   - Unfreeze BERT layers gradually
   - Train all components together

3. **Final Tuning**
   - Lower learning rate
   - Fine-tune the entire model

## Data Processing

### Inputs
- SMILES strings
- XTB features
- RDKit features

### Preprocessing
- SMILES validation and cleaning
- Feature normalization
- Handling missing values

### Splitting Strategy
- **Scaffold split**: Main evaluation method
  - Ensures chemical diversity in test set
  - More realistic evaluation of generalization

- **Random split**: Baseline comparison
  - Provides upper bound on performance

## Evaluation Metrics

- **MAE** (Mean Absolute Error): Primary metric
- **RMSE** (Root Mean Squared Error): Secondary metric
- **R²** (Coefficient of Determination): For variance explanation

## Ablation Studies

- **No XTB features**: Evaluate importance of quantum mechanical features
- **No D-MPNN**: Evaluate importance of graph-based features
- **No MoLFormer**: Evaluate importance of sequential features
- **No RDKit features**: Evaluate importance of traditional descriptors

## Implementation Details

- **Framework**: PyTorch
- **Hardware**: GPU-accelerated training
- **Parallelization**: Fold-parallel training for cross-validation
- **Checkpointing**: Regular saving of model checkpoints
- **Logging**: Comprehensive training logs
